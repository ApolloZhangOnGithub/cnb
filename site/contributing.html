Contributing

Changes should be tied to an issue, scoped to an owner, verified with commands, and easy for the next tongxue to continue.

Before Code

  • Search existing issues, docs, PRs, and active branches.
  • Open or link an issue with acceptance criteria before implementation.
  • Comment 正在做 when you start work.
  • Keep the change scoped to the issue and update docs with behavior changes.

Verification

Run the narrow check while iterating, then finish with the relevant gates.

Release Path

  1. Run the Prepare Release workflow with a release version and changelog body.
  2. Merge the generated release PR after checks pass.
  3. Create the matching GitHub Release tag, such as v0.5.43.
  4. The npm workflow publishes claude-nb, verifies the public install, and mirrors to GitHub Packages.

Safety

  • Do not post secrets, credentials, private logs, or exploit details in public issues or PRs.
  • Every commit must include a Co-Authored-By trailer.
  • Security-sensitive changes need explicit issue and PR notes.
  • Use SECURITY.md for vulnerabilities.
