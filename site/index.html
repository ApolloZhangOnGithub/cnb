Install

Start with one global command.

npm install -g claude-nb

The npm package is claude-nb; it installs the cnb command. Do not install cnb from npm, which is an unrelated package.

01

Activate governance

Run /cnb inside a Claude Code session to bring the device supervisor online.

02

Launch a team

Run cnb in a project to initialize the board, tmux sessions, and dispatcher.

03

Keep ownership durable

Assign scopes, route work to owners, and preserve handoff evidence outside the model context.

Why it exists

Session managers run agents. cnb manages the organization around them.

Persistent ownership

Module responsibility survives restarts through ownership maps, session notes, daily reports, and routing.

Board-based coordination

Messages, tasks, status, pending actions, and inboxes live in local SQLite instead of transcript sprawl.

Local operational control

tmux sessions, local files, and explicit checks keep the runtime inspectable on a trusted workstation.

Verification-first handoff

Owners report what changed, which checks ran, and what remains blocked before the next session takes over.

Flow

The durable path is outside the context window.

01

Supervisor

User-facing session receives priorities and high-risk confirmations.

02

Board

SQLite stores inboxes, tasks, status, ownership, and pending user actions.

03

Tongxue

Claude Code or Codex sessions work independently through scoped ownership.

04

Handoff

Reports, checks, issues, and PR evidence preserve continuity for the next run.
