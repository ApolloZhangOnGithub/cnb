# #168 Demo: 持续运维 — 7x24 监控、告警响应和轮班交接

**State:** OPEN
**Labels:** enhancement, phase:3

---

## 目标

展示 cnb 管理一个持续运行的运维任务：监控一个系统，自动响应告警，轮班交接，跨天持续运行。

## 为什么这是好 demo

- 展示 cnb 的长期运行能力（不是一次性任务）
- 轮班制是 cnb 交接机制的最佳展示场景
- incident ownership + SLA 是真实运维需求
- 最接近"AI 值班员"的想象

## 展示重点

- dispatcher 自动调度
- 告警 → bug report → 分配 → 修复 → 验证的完整闭环
- 轮班交接（daily report + shift handoff）
- Feishu 通知集成（告警推送到手机）
