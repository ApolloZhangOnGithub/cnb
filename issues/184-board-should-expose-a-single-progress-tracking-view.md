# #184 Board should expose a single progress tracking view

**State:** CLOSED

---

## Problem\nDogfooding showed that work is scattered across statuses, task queues, bugs, pending actions, and unread messages. Users cannot quickly tell which commitments are tracked or stalled without asking each session manually.\n\n## Acceptance\n- Provide one board command that summarizes active and pending tasks, open bugs, pending actions, unread messages, and per-session state.\n- Include active and pending task details so a supervisor can paste the output into Feishu.\n- Add regression coverage for the tracking view.
