# #64 20260508_组织体制改革提案（重点）

**State:** OPEN
**Labels:** phase:1, org-design, priority:p0

---

架构调整：
项目（repo）的负责人定为项目经理职位 
面向用户的claude code lead将不再分仓库，而是整个机器一个（暂成为机长同学或者user service buddy（usb同学））。如果用户一个不够同时聊可以开两个机长。整个机器设置成机长和副机长其实也挺好的，比较保险，也够用户聊，可以精心设计一下。不知道机器是否也需要一些机组成员，如果有的话有啥用，这个可能要想一下。然后用户默认cnb就是在全局的.cnb中创建一个机组来管理里面的cnb项目。
随着单台电脑甚至多台电脑的项目逐渐多起来，或许需要事业部、bu、事业群一类的概念，不过暂时还没有想好。
在手机等移动设备上应该有类似龙虾但不需要那么复杂的架构去实现一个移动设备的通讯，负责和各个机长对接交流，这样电脑上的机长就应该是日常跑着的。
对于mba/mbp等移动设备等，应该还用一种轻量化和敏捷的机组接力/空运机制，可以迅速的把维持转移到其他mac或云端系统上（现在云端买不了claude code cli，还有一些障碍。）。
然后通讯的架构，尤其是跨项目和跨终端，以及移动的通讯，首先考虑飞书。然后根据飞书的适应好权限是否适配、是否需要自主化等要求，可以考虑复用之前复现的微信App来开发（这个App目前仅实现了Mac电脑端的本地，不过界面还是很简洁的，内核也很轻盈）
其他还没想到。事情挺多了已经。
