# Code of Conduct

## Our Standard

cnb is a collaboration tool for human maintainers and AI tongxue. Contributors are expected to keep the project useful, inspectable, and respectful.

Good participation means:

- Be direct and constructive.
- Assume good intent, but ask for evidence when claims affect design, safety, or maintenance.
- Respect project terminology and contribution rules.
- Give credit for work, including AI-assisted work, through the required commit trailers.
- Keep disagreements focused on behavior, code, documentation, or project direction.

Unacceptable participation includes:

- Harassment, threats, insults, or targeted personal attacks.
- Sexualized, discriminatory, or demeaning language.
- Publishing private information without explicit permission.
- Repeatedly ignoring maintainer decisions, issue templates, security policy, or review boundaries.
- Using project systems to spam, impersonate contributors, or misrepresent work.

## Scope

This code applies in the repository, issue trackers, pull requests, project boards, chat spaces, demos, and any other official project venue.

## Enforcement

Maintainers may remove comments, close discussions, block contributors, revert changes, or restrict access when conduct disrupts the project or creates risk for users.

When possible, maintainers should explain the action taken and the behavior that needs to change. Severe or repeated violations may be handled without warning.

## Reporting

Report conduct concerns to the maintainers through a private GitHub channel when available. If the concern involves a security vulnerability, follow [SECURITY.md](SECURITY.md) instead of opening a public issue.
